1. Stack at a glance reference

Trigger

PowerShell 5.1 / 7+ on Windows host (or pwsh on Linux/macOS). Scripts: run-open-prs.ps1 (batch), run.ps1 (single PR), build.ps1 (image). run-local.ps1 has been removed; use run.ps1 -Build.

Container

OCI image reviewforge:<tag>. Base node:24-bookworm-slim. Pi (@earendil-works/pi-coding-agent@<PI_VERSION>) installed globally. Python 3.11 stdlib + jinja2. ENTRYPOINT = python3 -m reviewforge.

Source of truth

Python package reviewforge under src/. The package owns the CLI, the pipeline, the ADO helper, and all runtime behavior.

External surfaces

Azure DevOps REST (Bearer PAT), pi CLI (subprocess, read-only), git over ADO credentials (via GIT_ASKPASS_SCRIPT). The Python discover subcommand replaces az repos pr list; PowerShell may still use either.

Idempotency

prb:<sha1(file|line|severity|title|message)[:12]> marker, last line of comment. Stable contract; do not change without a CHANGELOG entry.

Tests

pytest. Coverage gate: --cov-fail-under=95. PowerShell scripts are orchestration-only; the Python side owns behavior and carries the test gate.

3. End-to-end flow explanation

run-open-prs.ps1→ az repos pr list→ dedupe + filter→ interactive pick→ run.ps1 × N→ docker run -d IMAGE→ python -m reviewforge→ reviewforge.cli→ 12 stages→ ADO comments

Side-channel: config & credentials

  • PowerShell wrappers do not auto-load .env. The user (or direnv/PSProfile) populates the live process env. Precedence: CLI param > env > .env > default.
  • run.ps1 forwards .env via docker run --env-file when the file exists. If absent, it builds a temp env file from the process snapshot. Secrets never hit a persistent file.
  • Pi's auth.json (~/.pi/agent/auth.json) is mounted into the container at /root/.pi/agent/auth.json so subscription OAuth just works. Read-write so refresh tokens update.
  • Strip in subprocess: ai/runner.py:_scrub_ado_env removes ADO_AUTH_TOKEN, ADO_MCP_AUTH_TOKEN, ADO_API_KEY from the pi process env. Defense in depth — the model never sees ADO creds.

5. PowerShell wrapper layer reference + how-to

Three scripts + a shared module. All thin: orchestrate only. No ADO logic.

FileResponsibilityCalls
run-open-prs.ps1Batch. Discovers active PRs via az or python -m reviewforge discover, dedupes, optionally prompts interactively, then loops run.ps1.az repos pr list --status active or reviewforge discover, build.ps1, run.ps1
run-open-prs-scheduled.ps1Thin shim. Adds Start-Transcript log per run. Unattended.run-open-prs.ps1
run.ps1Single PR. Picks runtime, mounts auth.json, builds docker run argv, spawns detached.container runtime
build.ps1Image build with PI_VERSION arg.docker build / podman build
common.psm1Helpers: Resolve-ScriptConfig, Show-InteractivePrompt, Get-ContainerRuntime, Get-ReviewerEnvFile.—

Key behaviours worth knowing

  • Required env for batch: ADO_ORGANIZATION, ADO_PROJECTS, ADO_TARGET_BRANCHES — no built-in defaults. Hard fail with a hint pointing at the missing key.
  • PS 7+ parallel discovery: ForEach-Object -ThrottleLimit (min(N, 4)) with common.psm1 re-imported inside each runspace (script-scope helpers don't survive the runspace boundary). PS 5.1 falls back to serial.
  • Dedupe by (project|repoId|prId) so cross-project repos with the same name don't collide.
  • Interactive prompt auto-enables when [Console]::IsInputRedirected is false. CI / scheduled runs disable it implicitly.
  • Container runs detached (-d --rm) by design. The spawn exit code is what the wrapper reports; the review's own success is observable only via the artifact tree.
  • Per-PR container name: review-pr-<PR_ID> (when CONTAINER_NAME unset). Pass -KeepContainer to inspect it after the run.
  • Artifact storage: ARTIFACT_PATH env → bind-mounted local directory. Otherwise a named volume reviewforge-artifacts is created and reused.

6. Container & Python entrypoint explanation

Image

  • Base: node:24-bookworm-slim. OS deps: ca-certificates, git, python3, python3-pip, ripgrep.
  • Python: only jinja2>=3.1 on top of stdlib.
  • Pi: installed globally via npm with --ignore-scripts. Pin via PI_VERSION build-arg (default 0.79.1).
  • CRLF strip: find ./scripts ./src -name '*.py' -exec sed -i 's/\r$//' {} +. A trailing \r on the shebang turns python3 into python3\r and fails at exec with 127.
  • Env: PYTHONPATH=/app/src, WORKSPACE=/workspace, PI_SKIP_VERSION_CHECK=1, PI_TELEMETRY=0.
  • ENTRYPOINT: python3 -m reviewforge. CMD: ["review"].

Entry sequence

  1. python -m reviewforge is a one-liner that adds src/ to sys.path and calls reviewforge.cli.main.
  2. cli.main defaults to review when no argv. run.ps1 never passes subcommand args (just env), so the container always lands on the full pipeline.
  3. cmd_review builds a Config from env (no CLI args inside the container), validates it, then run_full(cfg).

7. Pipeline stage contract reference

reviewforge.pipeline.stages.DEFAULT_PIPELINE is a fixed list. orchestrator.run_stages walks it; each stage is a Stage subclass with two methods:

class Stage:
    name: str
    def should_run(self, ctx: StageContext) -> bool: ...
    def run(self, ctx: StageContext) -> dict: ...

Each stage returns a StageResult with status OK / SKIPPED / FAILED. FAILED short-circuits. Stages may declare what they read/write via the artifact dataclass; they communicate downstream via ctx.<slot> attributes populated by the orchestrator.

Adding a stage (canonical procedure)

  1. Implement YourStage under pipeline/stages/your_stage.py.
  2. Append to DEFAULT_PIPELINE in pipeline/stages/__init__.py. Do not reorder existing entries.
  3. Add artifact name to ARTIFACT_NAMES in artifacts/manager.py. Add the slot to the Artifacts dataclass + as_dict.
  4. Add a test under tests/ instantiating the stage with a stub StageContext; assert artifact exists.
  5. Update docs/reference/pipeline.md and AGENTS.md §5.

8. AI runner & Pi subprocess explanation + reference

PiRunner (ai/runner.py) is the only code that touches the pi CLI. Every stage that needs the model goes through it.

Command shape

pi \
  [--no-session] [--session-id <id>] [--clear-session] \
  --no-context-files --no-extensions --no-skills \
  --no-prompt-templates --tools read,grep \
  --model <PI_MODEL> --thinking medium \
  --append-system-prompt <augmented.md> \
  -p "Process the task described in the system prompt. ..."

The full payload (system prompt + instruction + unified diff) goes to stdin. The runner captures stdout to a file and stderr for token-usage parsing.

Session reuse

  • Default: --session-id pr-<id>-review-<run_id> so the system prompt + diff + prior turns persist between stages and chunks. First stage pays the full context cost; subsequent stages send only the new instruction.
  • Disable via PI_SESSION_ENABLED=0 or --no-pi-session for deterministic reruns.
  • Reset via --pi-session-clear to start a fresh session under the same id (post-schema-change).
  • Repair call: on invalid JSON, runner retries in the same session asking only for JSON — no full context resend.

Defense in depth

  • _scrub_ado_env strips ADO_AUTH_TOKEN / ADO_MCP_AUTH_TOKEN / ADO_API_KEY from the subprocess env.
  • --no-context-files --no-extensions --no-skills --no-prompt-templates --tools read,grep pins Pi to read-only repo tools. The model can read and search; it cannot run shells or call ADO.
  • Prompt augmentation: per-runner temp dir copies the source prompt with the LANGUAGE directive appended. Cached by source path; lives for the runner's lifetime.

Timeouts

PI_TIMEOUT_SECS defaults to 600 seconds. On timeout, the runner raises SystemExit and the stage is recorded as FAILED. There is no retry loop.

9. ADO integration & posting explanation + reference

All ADO side effects live under src/reviewforge/ado/. The only stage that calls them in production is PostToAdoStage.

Idempotency contract (the stable part)

Dedupe key: sha1(file|line|severity|title|message)[:12]. Excluded (noisy): suggestion, contextBasis, confidence, severity_calibration, created_at, updated_at. Marker regex: ^prb:([a-zA-Z0-9]{6,32})$ on its own line, last line of the comment.

Breaking change warning
Adding a field to the dedupe key will cause every existing comment to be re-posted on the next run. Document in CHANGELOG.md and call it out in the PR description.

Posting decision per finding

  1. Title starts with Work item #N? → strip file/line, post as general PR comment.
  2. Else: post as inline file comment on the chosen file/line. If placement fails, fall back to file-level comment.
  3. Marker computed; if already present in any thread's first comment, skip.
  4. Vote: if VOTE_WAITING_ON ≤ max severity, cast waiting for author. Default is none (no vote).

Thresholds (env)

  • FAIL_ON — non-zero container exit if any finding meets/exceeds.
  • VOTE_WAITING_ON — same severity ladder (none|nit|minor|major|blocker). Default none.
  • POST_MIN_SEVERITY — drop findings below this. Default is minor in the posting helper (note: the Config dataclass declares none, but the helper reads env directly with a minor fallback).
  • REQUIRE_CONTEXT_FOR — drop findings lacking contextBasis at these severities.
  • DROP_LOW_CONFIDENCE — drop confidence == "low".
  • MAX_FINDINGS — hard cap posted.

Stale-comment reconciliation

After posting, the helper scans existing bot threads whose (file, line) is no longer in the current diff and appends a "🤖 stale — this finding no longer anchors..." comment. Disabled via ANNOTATE_STALE=0. Threads are never deleted or resolved.

10. Artifact contract reference

artifacts.manager.ARTIFACT_NAMES is the stable contract. The list is append-only — never rename, never remove. Order:

metadata.json, diff.patch, changed-files.json, commits.txt,
intent.json, context-plan.json, collected-context.json, context-digest.json,
candidate-findings.json, verified-findings.json, severity-findings.json,
final-findings.json, posted-comments.json, run-summary.json,
review-system.combined.md, work-items.json, threads.json

The dataclass Artifacts mirrors each name as a Path attribute; as_dict returns the map consumed by tests and runners. raw/ is created eagerly for per-chunk Pi outputs.

Run summary

run-summary.json records per-stage {name, status, started_at, duration_ms, details, token_usage}. artifacts.summary redacts tokens/secrets before write. Treat it as the first stop when triaging a failure.

11. Configuration & precedence reference

Single source of truth: Config dataclass in src/reviewforge/config.py. Constructors:

  • Config.from_env() — legacy, used by tests and direct callers.
  • Config.from_sources(cli, env=os.environ) — preferred. Layers CLI overrides on top of env with alias resolution.

Precedence everywhere: CLI flag > env var > .env > default.

Env-var aliases

The token slot accepts SYSTEM_ACCESSTOKEN, ADO_AUTH_TOKEN, ADO_MCP_AUTH_TOKEN, ADO_API_KEY (first hit wins). PR identity accepts both PR_ID and PR_URL; URL parses down to (org, project, repo, pr_id) when individual keys are absent.

12. Where to extend safely how-to

Add a stage

Subclass Stage, append to DEFAULT_PIPELINE, add to ARTIFACT_NAMES, add a test. See §7.

Change the prompt

Edit files under prompts/ or mount replacements with -v /path/prompt.md:/cfg/prompt.md -e REVIEW_PROMPT_PATH=/cfg/prompt.md. Keep the JSON output contract.

Change standards

Mount -v /path/standards.md:/cfg/standards.md -e REVIEW_STANDARDS_PATH=/cfg/standards.md. Custom standards are appended verbatim.

Custom comment format

Set COMMENT_TEMPLATE_PATH to a Jinja2 file. The formatter enforces the marker as the last line regardless of template content.

Change the dedupe key

Don't. If you must, document as a breaking change in CHANGELOG and expect every existing comment to re-post.

Post only

python -m reviewforge post --input final-findings.json --pr 12345 runs the post-only pipeline.

13. Hard rules (do not break) reference

  1. The package owns all runtime behavior. Container entrypoint is python -m reviewforge; legacy ADO subprocess is python -m reviewforge.ado.cli. New behavior goes in src/reviewforge/.
  2. The LLM must not call Azure DevOps directly. Pi runs read-only. All ADO REST/subprocess calls live in reviewforge.ado.* and are reachable only from PostToAdoStage (or its helper subprocess).
  3. Posting is idempotent. Every comment gets a prb:<key> marker. existing_bot_markers() must be scanned before posting.
  4. Do not edit ARTIFACT_NAMES lightly. Stable contract for artifacts/pr-<PR_ID>/runs/<RUN_ID>/*. Add new files at the end; never rename or remove an entry.
  5. Marker regex is anchored. ^prb:([a-zA-Z0-9]{6,32})$, last line of the comment body.
  6. Coverage gate is 95%. New code in src/reviewforge/ must come with tests.
  7. open-prs from the Python CLI is intentionally unsupported. It fails fast with a pointer to ./run-open-prs.ps1.
  8. No repo-level Node package. The Docker image installs Pi globally via npm. Hosts do not need to pre-fetch PR branches.

16. Known limits & follow-ups explanation

  • Comment resolution loops on later pushes are not implemented. The bot appends a stale annotation but never resolves or deletes its own threads.
  • In-container tool gating beyond the container boundary is out of scope.
  • Very-large-single-file diff truncation is only partially mitigated via CHUNK_TRIGGER_DIFF_BYTES / DISABLE_CHUNK_REVIEW.
  • Auto-discovery of active PRs from the Python CLI is unsupported by design; run-open-prs.ps1 is the batch path.
